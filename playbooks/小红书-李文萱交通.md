## 操作步骤

### 第一步：确认账号

- [ ] 登录后检查右上角账号显示是否为「海外（YX-INNN交通出行）」
- [ ] 如不是，需切换账号

### 第二步：进入基础报表

> ⚠️ 注意：登录后默认落在「推广」页面，需从侧边栏进入数据页面

- [ ] 点击左侧边栏「数据」→「基础报表」
- [ ] 也可直接在侧边栏找到「基础报表」入口点击

### 第三步：设置日期筛选

> ✅ 优先使用「昨天」快捷按钮，最简单准确

- [ ] 在日期筛选区域点击「昨天」按钮
- [ ] 等待页面数据刷新
- [ ] 确认日期显示为「昨天」的日期（如今天4月3日，则显示4月2日）

### 第四步：读取「数据明细」合计行

### 第五步：数据校验

> 写入选项前必做校验，防止读到空值或错误数据

- [ ] 消费 > 0（账户余额不足时消费=0，需确认）
- [ ] 点击率 ≈ 点击量 ÷ 展现量（允许舍入误差）
- [ ] 私信进线数、开口数、留资数 ≥ 0
- [ ] 合计行有「合计」字样，不是单条数据

### 第六步：写入飞书

使用 lark-cli 写入。创建 .bat 文件（如 `write_row_jtwz.bat`）：

```batch
@echo off
lark-cli sheets +write --spreadsheet-token DuyywOXW5iPUAbk2JWOcKq1RnSe --sheet-id nfuwzz --range "B{行号}:U{行号}" --values "[[消费,展现量,点击量,点击率小数,平均点击成本,千展费用,点赞,评论,关注,分享,互动量,互动成本,私信进线数,私信进线成本,私信开口数,私信开口成本,私信开口条数,私信留资数,私信留资成本,0]]"
```

**注意：**
- 点击率必须是**小数形式**（2.96% → 0.0296）
- U列（多转化人数）固定写0
- 用.bat批处理文件执行，不要直接在PowerShell运行lark-cli

### 第七步：验证

- [ ] 检查返回的 revision 号是否变化
- [ ] 可选：用 `lark-cli sheets +read` 读取写入范围确认

## 常见错误

| 错误 | 原因 | 解决 |
|------|------|------|
| 数据全为0 | 账户余额不足，没有投放消耗 | 确认日期和账户状态 |
| 找不到合计行 | 滚动不够，或数据未加载完 | 等待页面完全加载 |
| 私信数据缺失 | 读了侧边栏概览数据 | 必须到「数据明细」读合计行 |
| 写入范围错误 | 行号算错 | 目标行号 = Excel日期 - 46111 |
