## 操作步骤

- [ ] 浏览器打开 https://partner.xiaohongshu.com/partner/subAccount-list
- [ ] 检查页面是否有协议复选框

- [ ] 登录后检查是否有弹窗

### 第二步：找到子账户

- [ ] 在「子账号名称」搜索框输入**"包易二奢"**
- [ ] **关键：** 必须点击下拉列表中的对应选项（只输入文字不会过滤）

  > ⚠️ 如果普通 ref 点击无法触发，使用 JS evaluate：
  > ```
  > browser act --request "{\"kind\":\"evaluate\",\"fn\":\"() => { const items = Array.from(document.querySelectorAll('[class*=d-grid-item]'));
  > const target = items.find(el => el.textContent.includes(\\"包易\\"));

- [ ] 点击「查询」按钮
- [ ] 在列表中找到该账号

> ⚠️ 弹出框中的「聚光平台」选项是 `<div class="d-grid-item">`，不是标准 button 元素

- [ ] 如果 ref 点击无效，使用 JS evaluate 点击：
  ```
  browser act --request "{\"kind\":\"evaluate\",\"fn\":\"() => { const items = Array.from(document.querySelectorAll('[class*=d-grid-item]'));
  const target = items.find(el => el.textContent.trim() === '聚光平台');
  if (target) { target.click(); return 'clicked 聚光平台'; }

### 第八步：写入飞书（基础报表数据）

**Sheet ID：** KlQEFs

**日期值格式：** A列必须包含 `.1` 后缀（如 46114.1）

创建 .bat 文件（如 `write_row_baoyi.bat`）：

```batch
@echo off
lark-cli sheets +write --spreadsheet-token M4Spsd0CGh0hh5tyae4cXdhWnRb --sheet-id KlQEFs --range "KlQEFs!A{行号}:U{行号}" --values "[[{Excel日期}.1,{消费},{展现量},{点击量},{点击率},{平均点击成本},{千展费用},{点赞},{评论},{关注},{分享},{互动量},{互动成本},{私信进线数},{私信进线成本},{私信开口数},{私信开口成本},{私信条数},{留资数},{留资成本},0]]"

## 关键规则

| 规则 | 说明 |
|------|------|
| 包易≠大阪/交通 | 本 playbook 只用于包易子账号 |
| vSellerId 从 URL 获取 | 跳转后立即记录，用于后续步骤 |
| 基础报表写入 KlQEFs | 第一步数据写入主 sheet |
| 笔记报表覆盖「2026笔记投放数据」 | 第二步数据覆盖到子表 |
| 点击率÷100 | 平台显示5.37%，写入用0.0537 |
| 聚光平台用 JS 点击 | div 元素普通 ref 点击无效，需用 JS evaluate |

## 关键教训

1. **vSellerId 必须记录** — 跳转到聚光平台后立即从 URL 获取，用于直接访问各报表
2. **笔记报表下载按钮** — 在数据明细行最右边，找向下箭头图标
3. **下载文件位置** — 默认保存到浏览器下载目录，完成后要打开
4. **粘贴到正确子表** — 「2026笔记投放数据」是独立 sheet，不是 KlQEFs

## 常见错误

| 错误 | 原因 | 解决 |
|------|------|------|
| 找不到笔记报表下载按钮 | 滚动位置不对 | 确认在数据明细区域，找行右侧箭头图标 |
| 下载按钮点击无反应 | div 元素不是标准 button | 尝试 JS evaluate 点击 |
| 「2026笔记投放数据」子表不存在 | sheet 名不一致 | 先确认实际 sheet 名称和 ID |
| 数据粘贴后乱码 | Excel 格式问题 | 全选 → 设置单元格格式为文本 → 重新复制粘贴 |
| 基础报表和笔记报表混淆 | 两个不同的报表入口 | 基础报表=广告数据，笔记报表=笔记维度数据 |
