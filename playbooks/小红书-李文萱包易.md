## 关键规则

| 规则 | 说明 |
|------|------|
| **A列禁止写入** | 写入范围必须从B列开始，绝对不碰A列 |
| **包易表只有B→P** | 15列，与交通（21列）/大阪（19列）不同 |
| **vSellerId必须验证** | 跳转后必须确认 `6876255d8c7f5d0015870152` |
| **子账户名只有一个横杠** | YX-包易二奢入门陪跑，不是 YX-- |
| **点击率÷100** | 平台显示5.37%，写入用0.0537 |
| **聚光平台用JS点击** | div 元素普通 ref 点击无效，需用 JS evaluate |

## 关键教训

1. **A列绝对禁止写入** — 乱写A列会导致日期识别错误
2. **包易只有B→P（15列）** — 不要写Q/R/S/U列
3. **包易vSellerId是6876255...** — 不是大阪的685918...，必须验证
4. **大阪vSellerId是685918...** — 包易≠大阪，必须分开验证
5. **搜索框必须点击下拉选项** — 只输入文字不会过滤
6. **聚光平台选项是div元素** — 不是button，需用JS evaluate点击

## ⚠️ 最重要规则：只看主表合计行

基础报表页面有**两个表格**：

| 表格 | 位置 | 特征 | 处理方式 |
|------|------|------|---------|
| **主表（合计行）** | 上方，加粗行 | 含完整数据 | **只用这个** |
| **副表（明细行）** | 下方 | 显示「暂无内容」| **完全忽略** |

- 副表显示「暂无内容」**不代表合计行里这些字段也是0**
- **只读主表合计行**，不管副表显示什么
- 4月5日错误根源：看到副表为空，就误以为合计行的留资数也是0
